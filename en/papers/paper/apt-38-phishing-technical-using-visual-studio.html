			 -=-=] Introduction [=-=-
	=-=-==-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-=


	In this paper, we will discuss a technique that APT-38 used, also known as the Lazarus Group, a state-funded group associated with
	North Korea, which has an extensively documented history of targeting cybersecurity researchers.

	The technique that will be covered in this paper consists of infecting files with the .vcxproj extension, used in Visual Studio projects,
	by injecting custom code.

	In a practical scenario, consider a security researcher who downloads a .sln file from a project hosted on GitHub. When importing
	this file in Visual Studio, the seemingly harmless action of compiling and running the project can trigger the execution of the malicious code
	previously injected into the .vcxproj file. As a result, the user's machine gets infected.

	=-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-==-=-=
    		   -=-=] What are .sln files? [=-=-
	=-=-==-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-=


	.sln files, short for "Solution," are used in Visual Studio to organize and manage software projects.

	.sln files contain information about the solution structure, projects involved, and build configurations, making it easier
	for developers to work as a team and coordinate efforts on software projects.
