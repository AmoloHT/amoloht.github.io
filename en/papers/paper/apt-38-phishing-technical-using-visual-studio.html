		 -=-=] Introduction [=-=-
=-=-==-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-=


In this paper, we will discuss a technique used by APT-38, also known as the Lazarus Group, a state-funded group associated with North Korea that has a well-documented history of targeting cybersecurity researchers.

The technique that will be discussed in this paper involves infecting files with the .vcxproj extension, used in Visual Studio projects, by injecting custom code.

In a practical scenario, consider a security researcher who downloads an .sln file from a project hosted on GitHub. When importing this file into Visual Studio, the seemingly harmless action of compiling and running the project can trigger the execution of the malicious code previously injected into the .vcxproj file. As a result, the user's machine is infected.

=-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-==-=-=
	  -=-=] What are .sln files? [=-=-
=-=-==-=-==-=-=-=-=-=-=-=-=-=-==-=-=-=-=-=-=-=-=-=-=-=-=-=


.sln files, short for "Solution", are used in Visual Studio to organize and manage software projects.

.sln files contain information about the solution structure, the projects involved, and the build settings, making it easier for developers to work together and coordinate efforts on software projects.
